# Детальное описание создания раздела "Интерпретация данных исследований"

## Обзор процесса

Раздел "Интерпретация данных исследований" создается в файле DOCX, который генерируется через эндпоинт `POST /health-passport/generate` (строки 93-94 в `routes.go`). Этот раздел содержит результаты анализа медицинских файлов, загруженных для приема пациента.

---

## Этап 1: Получение файлов для анализа

**Файл:** `internal/services/health_passport_service.go`  
**Функция:** `generatePassportContent()` (строки ~1090-1238)

1. **Получение файлов из приема:**
   - Система получает список файлов, прикрепленных к приему (`appointmentID`)
   - Если файлов нет → устанавливается сообщение "Файлы для анализа не найдены" (локализованное)

2. **Подготовка контекста для анализа:**
   - Создается контекст `health_passport_generation` с информацией о поле пациента
   - Если известен пол пациента, добавляется `patient_gender:Мужской/Женской`
   - Это помогает AI правильно интерпретировать референсные значения анализов

3. **Создание запроса на анализ:**
   ```go
   analysisReq := &FileAnalysisRequest{
       AppointmentID: appointmentID,
       Files:         fileReferences,
       Language:      lang,
       Context:       contextWithGender,
       MaxFiles:      11,
   }
   ```

## Этап 2: Анализ файлов через OpenAI

**Файл:** `internal/services/file_analysis.go`  
**Функция:** `AnalyzeFilesEnhanced()` (строки ~298-987)

1. **Параллельная обработка файлов:**
   - Файлы обрабатываются параллельно (batch size = 8)
   - Каждый файл анализируется через OpenAI Vision API
   - Используется специальный промпт для паспорта здоровья (см. `internal/openai/prompts.go`, строки 632-720)

2. **Специальный промпт для паспорта здоровья:**
   - Формат: **клиентоориентированный** и медицински качественный
   - Требования:
     - НЕ упоминать названия файлов, технические детали
     - НЕ писать "дата не указана" - просто пропускать дату
     - НЕ использовать многоточия
     - Каждый документ анализируется ОТДЕЛЬНО
     - Правильно определять тип исследования из содержимого

3. **Формат ответа от AI:**
   ```
   [ТОЧНОЕ НАЗВАНИЕ ИССЛЕДОВАНИЯ] (дата, если указана)
   
   Ключевые находки: [все отклонения от нормы, патологии, важные показатели]
   
   Заключение: [1-2 предложения с основным выводом]
   ```

4. **Обработка нескольких анализов в одном файле:**
   - Если в файле несколько анализов (например, несколько ЭКГ или комбинация анализов)
   - Каждый анализ представляется ОТДЕЛЬНО в указанном формате
   - Разные анализы разделяются пустой строкой

## Этап 3: Форматирование результатов анализа

**Файл:** `internal/services/file_analysis.go`  
**Функция:** `formatAnalysisResultsForHealthPassport()` (строки 1237-1439)

1. **Разделение нескольких анализов:**
   - Функция `splitMultipleStudies()` разделяет текст на отдельные исследования
   - Ищет вхождения "Ключевые находки:" и "Заключение:" для разделения

2. **Обработка каждого файла:**
   - Для каждого успешно обработанного файла:
     - Если найдены структурированные анализы → разделяются и добавляются отдельно
     - Если структурированных анализов нет → извлекаются `findings` и `conclusion` через `extractFindingsAndConclusion()`
     - Определяется тип документа (`detectDocumentTypeFromAnalysis()`)
     - Извлекается дата (`extractDateFromAnalysis()`)

3. **Проверка на дубликаты:**
   - Перед добавлением каждого исследования проверяется, не является ли оно дубликатом
   - Сравнение по:
     - Точному совпадению нормализованных заголовков
     - Схожести содержимого (более 80% совпадения)
     - Схожести заголовков и содержимого (более 90% совпадения)
   - Дубликаты НЕ добавляются

4. **Объединение результатов:**
   - Все секции объединяются через `\n\n` (двойной перенос строки)
   - Результат сохраняется в переменную `filesAnalysis`


## Этап 6: Парсинг и добавление структурированного контента

**Файл:** `internal/utils/pdf/generator.go`  
**Функция:** `parseAndAddStructuredResearchContent()` (строки 2221-2626)

### 6.1. Предобработка контента

1. **Очистка:**
   - Убираются разделители `---`
   - Нормализуются переносы строк (`\r\n` → `\n`)
   - Убираются лишние пробелы в начале и конце

### 6.2. Поиск исследований (многоуровневый алгоритм)

#### Уровень 1: Regex-поиск структурированного формата

**Паттерн:** `(?i)([^\n]+(?:\([^)]+\))?)\s*\n\s*(?:Ключевые находки:|Key findings:)\s*([^\n]+(?:\n[^\n]+)*?)\s*\n\s*(?:Заключение:|Conclusion:)\s*([^\n]+(?:\n[^\n]+)*?)(?:\n|$)`

- Ищет паттерн: `Название → Ключевые находки: → Заключение:`
- Для каждого совпадения:
  - Извлекает название исследования
  - Очищает название (убирает даты в скобках, убирает текст после "Ключевые находки")
  - Нормализует находки и заключение (убирает лишние пробелы)
  - Форматирует: `Название\n\nКлючевые находки: ...\n\nЗаключение: ...`

#### Уровень 2: Поиск по названиям исследований

Если regex нашел мало исследований (< 4), ищет по ключевым словам:

**Паттерны названий:**
- `Электрокардиограмма`, `ЭКГ`, `EKG`, `ECG`
- `Компьютерная томография`, `КТ`, `CT`
- `Магнитно-резонансная томография`, `МРТ`, `MRI`
- `Эхокардиография`, `ЭхоКГ`, `EchoCG`
- `Ультразвуковое исследование`, `УЗИ`, `Ultrasound`
- `Анализ крови`, `Общий анализ крови`, `Биохимический анализ`
- `Рентгенография`, `Рентген`, `X-ray`

- Находит все позиции начала исследований
- Разделяет контент по этим позициям

#### Уровень 3: Разделение по тройным пустым строкам

- Ищет паттерн `\n\n\n+` (тройные и более пустые строки)
- Разделяет контент по этим разделителям

#### Уровень 4: Поиск по паттернам названий (короткие строки)

- Ищет паттерн: `\n\n+` + короткая строка (до 100 символов)
- Исключает строки, начинающиеся с "Ключевые находки" или "Заключение"
- Разделяет контент по найденным позициям

#### Уровень 5: Разделение по "Заключение:"

- Ищет все вхождения "Заключение:" или "Conclusion:"
- Разделяет контент по этим маркерам

#### Уровень 6: Построчный парсинг

- Разбивает текст на строки
- Отслеживает секции: название → находки → заключение
- Сохраняет исследование при обнаружении "Заключение:"

#### Уровень 7: Поиск по типам исследований (ключевые слова)

- Ищет паттерны начала исследований (ЭКГ, КТ, МРТ, УЗИ и т.д.)
- Разделяет контент по найденным позициям

#### Уровень 8: Fallback

- Если ничего не найдено → весь контент обрабатывается как одно исследование

### 6.3. Удаление дубликатов

**Функция:** `removeDuplicateStudies()` (не показана в коде, но вызывается на строке 2597)

- Сравнивает названия исследований
- Удаляет дубликаты перед добавлением в документ

### 6.4. Парсинг каждого исследования

**Функция:** `parseSingleStudy()` (строки 2807-3055)

Для каждого найденного исследования:

1. **Извлечение структуры:**
   - Название исследования (первая строка или до "Ключевые находки:")
   - Ключевые находки (после "Ключевые находки:")
   - Заключение (после "Заключение:")

2. **Очистка названия:**
   - Убираются markdown заголовки (`#`, `##`, `###`)
   - Убирается жирный текст (`**текст**`)
   - Убираются даты в скобках в конце
   - Проверяется длина (не должно быть слишком коротким)

3. **Определение секций:**
   - Если строка начинается с "Ключевые находки:" → секция `findings`
   - Если строка начинается с "Заключение:" → секция `conclusion`
   - Иначе → накопление названия исследования

4. **Обработка неструктурированного формата:**
   - Если не найдены маркеры "Ключевые находки:" и "Заключение:"
   - Используется `extractStudyNameFromLine()` для извлечения названия
   - Остальной текст идет в находки

5. **Добавление в документ:**
   - Название исследования: жирный, размер 12pt
   - Ключевые находки: обычный текст, размер 11pt
   - Заключение: обычный текст, размер 11pt
   - Между исследованиями добавляется пустая строка

## Этап 7: Финальное форматирование в DOCX

**Файл:** `internal/utils/pdf/generator.go`  
**Функция:** `parseSingleStudy()` → добавление в документ (строки 3056-3134)

1. **Добавление названия исследования:**
   ```go
   para := doc.AddParagraph()
   run := para.AddRun()
   run.AddText(studyName)
   run.Properties().SetBold(true)
   run.Properties().SetSize(12 * measurement.Point)
   ```

2. **Добавление ключевых находок:**
   ```go
   para = doc.AddParagraph()
   run = para.AddRun()
   run.AddText("Ключевые находки: " + findings)
   run.Properties().SetSize(11 * measurement.Point)
   ```

3. **Добавление заключения:**
   ```go
   para = doc.AddParagraph()
   run = para.AddRun()
   run.AddText("Заключение: " + conclusion)
   run.Properties().SetSize(11 * measurement.Point)
   ```

4. **Выравнивание:**
   - Все параграфы выравниваются по ширине (`ST_JcBoth`)
   - Шрифт: Arial 11pt (для текста), 12pt (для названий)

## Особенности и важные моменты

1. **Многоуровневый парсинг:** Система использует 8 уровней поиска исследований для максимальной надежности

2. **Обработка дубликатов:** Автоматическое удаление дубликатов на основе схожести заголовков и содержимого

3. **Гибкость формата:** Система может обработать как структурированный формат (с маркерами "Ключевые находки:" и "Заключение:"), так и неструктурированный текст

4. **Очистка данных:** Автоматическое удаление markdown, дат, лишних пробелов для чистого отображения

5. **Локализация:** Поддержка русского, казахского и английского языков

6. **Обработка ошибок:** Если анализ файлов не удался, раздел все равно создается с сообщением об ошибке (анализ файлов опционален)

7. **Витальные показатели:** Параллельно с анализом извлекаются витальные показатели (температура, давление, пульс, сатурация) и обновляются в данных пациента
